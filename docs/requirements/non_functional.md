# MindFlow 要件定義書 - 非機能要件

更新日: 2026-04-06

---

## 1. 概要

本書は MindFlow web アプリケーションの非機能要件を定義する。
品質特性（ISO 25010 ベース）に基づき、MECE に分類する。

### 1.1 要件分類体系

```
NFR: 非機能要件
├── NFR-PER: 性能
├── NFR-REL: 信頼性
├── NFR-USA: 使用性
├── NFR-SEC: セキュリティ
├── NFR-MNT: 保守性
├── NFR-PRT: 移植性
├── NFR-OPS: 運用性
├── NFR-SCA: スケーラビリティ
└── NFR-DAT: データ管理
```

## 2. NFR-PER: 性能要件

### NFR-PER-001: Web API 応答時間

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-PER-001 |
| 要件名 | API レスポンス時間 |
| 説明 | FastAPI エンドポイントのレスポンス時間 |
| 基準 | アイテム 1000 件以下で 200ms 以内（P95） |
| 優先度 | 必須 |

### NFR-PER-002: データベースクエリ性能

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-PER-002 |
| 要件名 | データベースクエリのレスポンス時間 |
| 説明 | PostgreSQL クエリのレスポンス時間 |
| 基準 | 単一クエリで 50ms 以内（P95） |
| 実現手段 | 接続プーリング（pool_size=5, max_overflow=10）、クエリ最適化 |
| 優先度 | 必須 |

### NFR-PER-003: 静的ファイル配信

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-PER-003 |
| 要件名 | 静的ファイルサイズと配信速度 |
| 説明 | CSS、JS、画像などの総サイズと配信速度を最適化 |
| 基準 | CSS/JS の合計約 108KB 以内、配信時間 100ms 以内 |
| 実現手段 | gzip 圧縮、ブラウザキャッシュ（Cache-Control ヘッダ） |
| 優先度 | 推奨 |

### NFR-PER-004: ページ初期表示時間

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-PER-004 |
| 要件名 | ページ初期表示時間（FCP） |
| 説明 | ユーザーが最初のコンテンツを見るまでの時間 |
| 基準 | 2 秒以内 |
| 優先度 | 推奨 |

### NFR-PER-005: HTMX によるインタラクション

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-PER-005 |
| 要件名 | ページ遷移なしのインタラクション応答 |
| 説明 | HTMX を用いた DOM 部分更新による即座なフィードバック |
| 基準 | 部分更新応答時間 300ms 以内 |
| 優先度 | 必須 |

## 5. NFR-SEC: セキュリティ要件

### NFR-SEC-001: パスワード暗号化

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-001 |
| 要件名 | パスワードの安全な保存 |
| 説明 | ユーザーパスワードを bcrypt で ハッシュ化して保存 |
| 基準 | bcrypt round=12 以上 |
| 優先度 | 必須 |

### NFR-SEC-002: セッション管理

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-002 |
| 要件名 | 安全なセッションクッキー設定 |
| 説明 | ユーザーセッションを セキュアなクッキーで管理 |
| クッキー設定 | HttpOnly=true, Secure=true, SameSite=Lax, Max-Age=86400 (24h) |
| 優先度 | 必須 |

### NFR-SEC-003: CSP ヘッダー

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-003 |
| 要件名 | Content Security Policy (CSP) ヘッダー |
| 説明 | XSS 攻撃を防ぐため CSP ヘッダーを設定 |
| 基準 | default-src 'self', script-src 'self' 'unsafe-inline', style-src 'self' 'unsafe-inline' |
| 優先度 | 必須 |

### NFR-SEC-004: X-Frame-Options ヘッダー

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-004 |
| 要件名 | クリックジャッキング対策 |
| 説明 | X-Frame-Options ヘッダーで iframe 埋め込みを禁止 |
| 基準 | X-Frame-Options: DENY |
| 優先度 | 必須 |

### NFR-SEC-005: HSTS ヘッダー

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-005 |
| 要件名 | HTTPS 強制（HSTS） |
| 説明 | Strict-Transport-Security ヘッダーで HTTPS 通信を強制 |
| 基準 | Strict-Transport-Security: max-age=31536000; includeSubDomains |
| 優先度 | 必須 |

### NFR-SEC-006: 多テナント データ分離

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-006 |
| 要件名 | ユーザーデータの完全分離 |
| 説明 | 各ユーザーは自分のデータのみアクセス可能 |
| 実現手段 | リポジトリ層で user_id フィルタリング、全データベースクエリに user_id 条件を付与 |
| 優先度 | 必須 |

### NFR-SEC-007: レート制限

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-007 |
| 要件名 | ブルートフォース攻撃対策 |
| 説明 | ログイン試行回数を制限 |
| 基準 | 同一 IP から 5 回の失敗後、5 分間ブロック |
| 実現手段 | SlowAPI またはカスタムミドルウェア |
| 優先度 | 必須 |

### NFR-SEC-008: 入力値検証

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-008 |
| 要件名 | ユーザー入力の検証・サニタイズ |
| 説明 | すべてのユーザー入力を検証し、SQLインジェクション、XSS を防止 |
| 実現手段 | Pydantic モデルによるスキーマ検証、SQLAlchemy パラメータクエリ |
| 優先度 | 必須 |

### NFR-SEC-009: タイミング攻撃対策

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-009 |
| 要件名 | パスワード検証のタイミング攻撃対策 |
| 説明 | bcrypt 検証時間が一定になるよう実装 |
| 実現手段 | bcrypt.checkpw() の使用（恒時間実装） |
| 優先度 | 推奨 |

### NFR-SEC-010: 機密情報の非保存

| 項目 | 内容 |
|------|------|
| 要件 ID | NFR-SEC-010 |
| 要件名 | 機密情報のコード内非保存 |
| 説明 | API キー、パスワード、DB URL などを環境変数で管理 |
| 実現手段 | .env ファイル（.gitignore 対象）、環境変数読み込み |
| 優先度 | 必須 |

## 補足: 要件の追跡・更新

各非機能要件は以下のプロセスで管理される：

1. **実装フェーズ**: 設計・開発時に該当要件を確認
2. **テストフェーズ**: テストケース・テスト計画で要件カバレッジを確認
3. **レビューフェーズ**: PR コメント・レビューで要件合致性を検証
4. **リリースフェーズ**: リリースノートで対応状況を記録

要件変更時は本ドキュメントを更新し、git history で変更の追跡を残すこと。
