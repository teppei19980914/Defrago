# リリース前チェックプロセス

**コミット＆プッシュ前に、以下3つのチェックを必ず実施すること。**
このプロセスはすべてのプログラム変更に適用される。

## 1. セキュリティチェック

現代の攻撃手法（ディレクトリトラバーサル、XSS、CSRF、SQLインジェクション等）に対応できるアプリを目指す。

### チェック項目

| カテゴリ | チェック内容 |
|---------|------------|
| **XSS** | テンプレートで `|safe` や `{% autoescape false %}` を使用していないか |
| **SQLインジェクション** | ORMのパラメータ化クエリを使用しているか（文字列結合でSQL構築していないか） |
| **CSRF** | POSTフォームが適切に保護されているか（SameSite Cookie等） |
| **入力バリデーション** | ユーザー入力にバリデーション（型・長さ・形式）を適用しているか |
| **認証・認可** | 保護が必要なエンドポイントに `require_auth` が設定されているか |
| **セキュリティヘッダー** | CSP, X-Frame-Options, HSTS等が設定されているか |
| **パスワード** | bcrypt等の安全なハッシュアルゴリズムを使用しているか |
| **機密情報** | SECRET_KEY等がハードコードされていないか |

### 実行手順

```
1. 変更したファイルのユーザー入力処理を確認
2. テンプレートのエスケープ処理を確認
3. 新規エンドポイントの認証設定を確認
4. 問題があれば修正してからコミット
```

## 2. パフォーマンスチェック

パフォーマンスはユーザーの離脱に直接影響する。ボトルネックを解消し、サクサク動くアプリを目指す。

### チェック項目

| カテゴリ | チェック内容 |
|---------|------------|
| **DBクエリ** | N+1問題が発生していないか。不要な全件取得をしていないか |
| **データ構造** | ルックアップにO(1)のdict/setを使用しているか（リスト線形探索を避ける） |
| **ミドルウェア** | 静的ファイルリクエストで不要な処理をスキップしているか |
| **テンプレート** | 不要な再計算・再描画を避けているか |
| **静的アセット** | キャッシュバスティング用のバージョンパラメータが付与されているか |

### 実行手順

```
1. 変更したルーターのDB呼び出しパターンを確認
2. ループ内での非効率な処理がないか確認
3. HTMX応答で最小限のデータのみ返しているか確認
```

## 3. 単体テスト

最低限機能として動作するものをコミットし、デプロイ後に公開する。

### 実行コマンド

```bash
# lint + format + テストの全チェック
uv run ruff check . && uv run ruff format --check . && uv run pytest
```

### 合格基準

- **lint**: エラーゼロ
- **format**: 差分ゼロ
- **テスト**: 全件パス
- **カバレッジ**: 新規コードは100%カバレッジを目指す

## チェックプロセスフロー

```
コード修正完了
    ↓
┌─────────────────────────────────┐
│ 1. セキュリティチェック          │
│    - 入力バリデーション確認      │
│    - XSS/CSRF対策確認            │
│    - 認証設定確認                │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│ 2. パフォーマンスチェック        │
│    - DBクエリ効率確認            │
│    - 不要な処理の排除            │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│ 3. 単体テスト                    │
│    - lint / format / pytest      │
│    - 全件パス確認                │
└─────────────────────────────────┘
    ↓
  全チェック合格 → コミット＆プッシュ
```
